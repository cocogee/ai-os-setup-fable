# 06_skills — Repeatable capabilities

**Type:** Capability layer.

**Belongs here:** One markdown file per skill. A skill is a saved, reusable set of instructions for a task you do repeatedly.

**Does not belong here:** One-off prompts, project content, scripts with external side effects (those need safety review first — see SAFETY_RULES.md).

**Before modifying:** Skills improve through use. After running a skill, ask the user what to keep/change, then update the skill file and note it in 00_system/CHANGELOG.md.

**Starter skills:** grill_me, update_context, create_project_brief, weekly_review, verify_output.
